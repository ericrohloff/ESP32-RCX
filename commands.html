Setup

The ESP32 acts as an IR tower. Install the driver once, then import it in every script you write.

  1. Connect to the ESP32 using the serial terminal in the app.
  2. Click Install RCX Lib — this uploads rcx_driver.py to the ESP32 filesystem.
  3. Write your script using the commands below, then click run as main.py to flash and run it. 
from rcx_driver import rcx   # always start with this line

rcx.beep()                    # quick test — you should hear a beep
Tip: Use the Code Templates panel on the main page to load ready-made starter programs into the editor.

Program Download (Store on RCX)

Instead of continuously sending IR commands, you can download programs directly to the RCX's memory and execute them standalone. This is more reliable and allows the RCX to run programs without the ESP32 connection.

How It Works

  1. Create a program using ProgramBuilder
  2. Download it to one of 8 program slots on the RCX
  3. Execute the program anytime using its slot number
  4. Programs persist even after RCX power cycles

Quick Example

from rcx_driver import rcx, ProgramBuilder

# Create and download a program
prog = ProgramBuilder("beep_sequence")
prog.beep()
prog.beep(sound=3)
prog.beep(sound=5)
prog.download(rcx, slot=0)

# Run it anytime
rcx.execute_program(slot=0)

Program Management

Available methods for program control:

program
prog.download(rcx, slot=0)

Download a program to RCX memory slot (0-7).

program
rcx.execute_program(slot=0)

Execute a stored program by slot number.

program
rcx.stop_program()

Stop the currently running program.

program
rcx.delete_program(slot=0)

Delete a program from RCX memory.

Building Programs with Commands

from rcx_driver import rcx, ProgramBuilder

prog = ProgramBuilder("motor_test")

# Chain methods for clean syntax
prog.set_power(0, 5) \
    .set_power(1, 5) \
    .motor_on(0) \
    .motor_on(1)

# Add beep
prog.beep(sound=3)

# Download and execute
prog.download(rcx, slot=0)
rcx.execute_program(0)
Why Program Download?
  • No IR connection needed after download
  • Programs run on RCX independently
  • Faster and more reliable
  • Store up to 8 different programs

High-Level Motion Commands

These commands control motors A and B together and handle timing automatically. They are the easiest way to make a two-motor robot move.

motion
rcx.move(speed=7, duration=None)

Drive forward on motors A and B.

speed 0–7  |  duration seconds, or omit to run until stop() 

rcx.move(speed=5, duration=2.0)
# moves 2 s then auto-stops
motion
rcx.backward(speed=7, duration=None)

Drive in reverse on motors A and B.

rcx.backward(speed=4, duration=1.5)
motion
rcx.turn_left(speed=7, duration=None)

Pivot left: motor A forward, motor B reverse.

rcx.turn_left(duration=0.5)
motion
rcx.turn_right(speed=7, duration=None)

Pivot right: motor A reverse, motor B forward.

rcx.turn_right(duration=0.5)
motion
rcx.stop()

Coast all three motors to a stop (float).

rcx.stop()
motion
rcx.brake()

Hard-brake all three motors immediately.

rcx.brake()

Individual Motor Commands

Use these when you need to control motors A, B, or C independently. Motor IDs: 0 = A, 1 = B, 2 = C.

motor
rcx.motor_on(motor_id, direction=0)

Turn on a single motor.

direction: 0 = forward, 1 = reverse

rcx.motor_on(0)           # Motor A forward
rcx.motor_on(2, direction=1)  # Motor C reverse
motor
rcx.motor_off(motor_id)

Coast (float) a single motor.

rcx.motor_off(0)   # coast Motor A
motor
rcx.motor_brake(motor_id)

Hard-brake a single motor.

rcx.motor_brake(1)   # brake Motor B
motor
rcx.set_power(motor_id, power)

Set power level before calling motor_on().

power: 0–7

rcx.set_power(0, 4)   # Motor A at half power
rcx.motor_on(0)

Sound

sound
rcx.beep(sound=2)

Play one of the RCX's built-in sounds.

sound=1 blip
sound=2 beep (default)
sound=3 sweep
sound=4 pling
sound=5 buzz 
rcx.beep()          # default beep
rcx.beep(sound=4)   # pling

Utilities

util
rcx.wait(seconds)

Pause the script for the given number of seconds.

rcx.wait(1.5)   # pause 1.5 seconds
util
rcx.ping()

Send an alive-check to the RCX. Returns (True, b"") if the RCX responds. Use this to verify IR is working. 

ok, _ = rcx.ping()
print("alive" if ok else "no response")

Example Programs

Square drive

from rcx_driver import rcx

for _ in range(4):
    rcx.move(speed=6, duration=1.5)    # straight
    rcx.turn_right(speed=5, duration=0.6)  # ~90 °

rcx.stop()
rcx.beep()

Motor power ramp

from rcx_driver import rcx

for power in range(8):           # 0 → 7
    rcx.set_power(0, power)
    rcx.motor_on(0)
    rcx.wait(0.4)

rcx.motor_off(0)
rcx.beep()

Back and forth

from rcx_driver import rcx

for _ in range(3):
    rcx.move(speed=7, duration=1.0)
    rcx.backward(speed=7, duration=1.0)

rcx.stop()
rcx.beep()
